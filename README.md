# Y-Agent Research Blog

Research blog for Zhuoran Yang's group, built with [Hugo](https://gohugo.io/) and the [Dream](https://github.com/g1eny0ung/hugo-theme-dream) theme.

**Live site:** https://Y-Agent.github.io/

## Repository Structure

This repo uses **two branches**:

| Branch | Purpose |
|--------|---------|
| `source` | Hugo source files (markdown, templates, config). **All edits go here.** |
| `main` | Compiled HTML output. **Never edit directly** — auto-generated by GitHub Actions. |

When you push to `source`, a GitHub Actions workflow automatically builds the site with Hugo and deploys the output to `main`.

### Directory Layout (on `source` branch)

```
.
├── .github/workflows/gh-pages.yml   # CI/CD pipeline
├── hugo.toml                         # Site configuration
├── content/posts/                    # Blog posts (Markdown)

```

## Git Workflow for Adding a New Blog Post

Follow these steps to add a new post using proper Git workflow.

### Step 1: Clone and Set Up

```bash
git clone https://github.com/Y-Agent/Y-Agent.github.io.git
cd Y-Agent.github.io
git checkout source
```

### Step 2: Create a Feature Branch

Always create a new branch off `source` for your work:

```bash
git checkout source
git pull origin source
git checkout -b blog/your-post-name
```

Use a descriptive branch name, e.g., `blog/modular-addition` or `blog/icl-regression`.

### Step 3: Add Your Blog Post

There are **two approaches** depending on your content:

---

#### Approach A: Markdown Post (Recommended)

Used by: `in_context_regression.md`, `modular_addition_feature_learning.md`

Best for posts written in Markdown with LaTeX math, inline HTML styling, and images.

**1. Create the Markdown file:**

```bash
# Create your post file
touch content/posts/your_post_name.md
```

**2. Add Hugo frontmatter** at the top of the file:

```yaml
---
title: "Your Paper Title Here"
date: 2025-06-18
author: "Author One, Author Two, Author Three"
cover: /images/your_post_name/cover_image.png
categories:
  - "Research Blog"
tags:
  - "tag1"
  - "tag2"
draft: false
math: true
toc: true
---
```

Required frontmatter fields:
- `title`: The post title shown on the blog
- `date`: Publication date in `YYYY-MM-DD` format
- `author`: Comma-separated author names
- `cover`: Path to the cover image (shown on the blog homepage card)
- `categories`: Use `"Research Blog"` to match existing posts
- `tags`: Relevant topic tags
- `draft`: Set to `false` to publish, `true` to hide
- `math`: Set to `true` if the post contains LaTeX equations
- `toc`: Set to `true` to enable the Table of Contents sidebar

**3. Copy images** to the static directory:

```bash
mkdir -p static/images/your_post_name/
cp /path/to/your/images/* static/images/your_post_name/
```

**4. Reference images** in your Markdown using absolute paths from the site root:

```html
<p align="center">
<img src="/images/your_post_name/figure1.jpg" width="60%">
</p>
```

For **multiple images side by side**, use a flex container (percentage widths alone don't work reliably in the Hugo theme's content area):

```html
<div style="display: flex; justify-content: center; gap: 4px; flex-wrap: nowrap;">
<img src="/images/your_post_name/fig_a.jpg" style="width: 48%; min-width: 0;">
<img src="/images/your_post_name/fig_b.jpg" style="width: 48%; min-width: 0;">
</div>
```

**5. Add post metadata** (authors, links) below the frontmatter. Follow the existing format:

```html
<strong style="font-size: 0.85em; letter-spacing: 1px; color: #3B9DD9;">AUTHORS:</strong>
<small>
<a href="https://author1.github.io/">Author One</a>,
<a href="https://author2.github.io/">Author Two</a>
</small>

<strong style="font-size: 0.85em; letter-spacing: 1px; color: #3B9DD9;">AFFILIATIONS:</strong>
<small>Department of Statistics and Data Science, Yale University</small>

<strong style="font-size: 0.85em; letter-spacing: 1px; color: #3B9DD9;">LINKS:</strong>
<span style="font-size: 0.9em;">
<strong style="color: #D98C3B;">arXiv:</strong> <a href="https://arxiv.org/abs/XXXX.XXXXX" target="_blank"><strong>Paper</strong></a> &nbsp;|&nbsp;
<strong style="color: #4CAF7D;">GitHub:</strong> <a href="https://github.com/Y-Agent/your-repo" target="_blank"><strong>Code</strong></a> &nbsp;|&nbsp;
<strong style="color: #A96CD5;">Demo:</strong> <a href="https://huggingface.co/spaces/..." target="_blank"><strong>HuggingFace Space</strong></a>
</span>
```

#### Approach B: Static HTML Post (with Redirect)

Used by: `taming-sae-gba.md`

Best for posts that are pre-built HTML pages (e.g., exported from a web framework or custom HTML/CSS/JS).

**1. Place your HTML files** in the `static/` directory:

```bash
mkdir -p static/your-post-name/
cp /path/to/your/index.html static/your-post-name/
cp -r /path/to/your/assets/ static/your-post-name/
```

Files in `static/` are copied as-is to the site root. So `static/your-post-name/index.html` becomes accessible at `https://Y-Agent.github.io/your-post-name/index.html`.

**2. Create a Markdown stub** in `content/posts/` that redirects to your HTML page:

```yaml
---
title: "Your Paper Title"
date: 2025-06-18
author: "Author One, Author Two"
cover: "/images/your-post-name/cover.png"
categories:
  - "Research Blog"
tags:
  - "tag1"
draft: false
---

<script>
  window.location.href = "/your-post-name/index.html";
</script>

<noscript>
  <meta http-equiv="refresh" content="0;url=/your-post-name/index.html">
  <p>Please click <a href="/your-post-name/index.html">here</a> to view the post.</p>
</noscript>
```

The Markdown stub creates a card on the blog homepage. When clicked, it immediately redirects to your standalone HTML page.

**3. Also add a summary** below the redirect script (this content appears in search results and RSS, even though users are redirected):

```markdown
## Abstract
Your abstract text here...
```

---

### Step 4: Commit and Push

```bash
git add content/posts/your_post_name.md static/images/your_post_name/
git commit -m "Add new blog post: Your Post Title"
git push origin blog/your-post-name
```

### Step 5: Create a Pull Request and Merge

```bash
# Create a PR targeting the source branch
gh pr create --base source --title "Add blog: Your Post Title" --body "New research blog post on ..."

# After review, merge the PR into source
gh pr merge --squash
```

Alternatively, merge via the GitHub web UI.

Once merged into `source`, the GitHub Actions workflow automatically builds and deploys the site. The post will be live within a few minutes.

### Step 6: Verify

Visit https://Y-Agent.github.io/ and confirm your post appears on the homepage and renders correctly.

## LaTeX / MathJax: Important Notes

This site uses **MathJax 3.2.2** for LaTeX rendering. There is a critical configuration that ensures math formulas work correctly.

### The Goldmark Passthrough Fix

Hugo's Markdown processor (goldmark) normally interprets `_` as italic/emphasis markers. This **breaks LaTeX formulas** that contain underscores, such as:

```latex
$$\underbrace{\cos(\omega_k x)}_{\text{signal}}$$
```

Without the fix, goldmark would process `_{\text{signal}}` as italic text, corrupting the equation before MathJax ever sees it.

**The fix** is the `passthrough` extension in `hugo.toml`:

```toml
[markup.goldmark.extensions.passthrough]
  enable = true
  [markup.goldmark.extensions.passthrough.delimiters]
    block = [['$$', '$$'], ['\[', '\]']]
    inline = [['$', '$'], ['\(', '\)']]
```

This tells goldmark: **do not process any Markdown syntax inside math delimiters**. Everything between `$...$`, `$$...$$`, `\(...\)`, and `\[...\]` is passed through verbatim to MathJax.

### Do NOT remove or modify this configuration. It is essential for all math rendering on the site.

### LaTeX Best Practices

1. **Inline math:** Use `$...$` or `\(...\)`.

   ```markdown
   The frequency is $\omega_k = 2\pi k / p$.
   ```

2. **Display math:** Use `$$...$$` or `\[...\]`.

   ```markdown
   $$
   \sum_{k=1}^{n} k^2 = \frac{n(n+1)(2n+1)}{6}
   $$
   ```

3. **Aligned equations:** Use `\begin{align}...\end{align}` inside `$$`.

4. **Underscores in math are safe** thanks to the passthrough fix — you can write `\underbrace{x}_{y}`, `\sum_{i=1}^{n}`, `\mathcal{N}_k`, etc. without escaping.

5. **Escaping underscores outside math:** If you use underscores in regular Markdown text and they get interpreted as italic, escape them: `\_`.

6. **Frontmatter:** Always set `math: true` in posts that contain LaTeX.

### Supported LaTeX Packages

The MathJax configuration loads: `base`, `ams`, `noerrors`, `noundefined`, `newcommand`, `boldsymbol`.

### Layout Details

The blog post layout (`single.html`) uses:
- **Flexbox** with content taking all available width and a fixed **220px TOC sidebar** on the right
- `max-w-none` on the prose container to remove the default width cap
- The TOC is **sticky** (stays visible while scrolling)

## Troubleshooting

### Math equations not rendering

1. Check that `math: true` is in your post's frontmatter
2. Verify the passthrough config exists in `hugo.toml` (see LaTeX section above)
3. Check browser console for MathJax errors
4. Common issue: using `_` inside markdown text (outside math delimiters) — escape with `\_`

### Images not showing

1. Images must be in `static/images/your_post_name/`
2. Reference with absolute paths: `/images/your_post_name/figure.jpg`
3. Do NOT use relative paths like `./images/figure.jpg` — they won't resolve correctly in Hugo

### Side-by-side images stacking vertically

Use the flex container pattern instead of `<p align="center">` with percentage widths:

```html
<div style="display: flex; justify-content: center; gap: 4px; flex-wrap: nowrap;">
<img src="/images/..." style="width: 48%; min-width: 0;">
<img src="/images/..." style="width: 48%; min-width: 0;">
</div>
```

### Build fails on GitHub Actions

1. Check the Actions tab on GitHub for error logs
2. Common cause: invalid TOML in `hugo.toml`
3. Test locally first: `hugo server -D` (requires Hugo installed)

### Post not appearing on homepage

1. Check `draft: false` in frontmatter
2. Check `date` is not in the future
3. Verify the file is in `content/posts/` (not `content/` root)
